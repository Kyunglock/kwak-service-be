## S — 어떤 서비스였나

<!-- 앵커 1~2문장 + 등장인물 소개 + 문제 상황. "그런데…"로 T장에 연결 -->

투자 도우미 서비스를 만들고 있다. 본체 서버(portal) 하나가 로그인부터
포트폴리오 관리까지 다 하고, 그중 AI 기능이 하나 있다 — 사용자의
포트폴리오를 로컬 LLM이 분석해 리포트를 만들어주는 **인사이트**다.

처음엔 portal이 LLM을 직접 호출했다. 그런데…

<!-- 분리 전 미니 구조도: 사용자 → portal → (직접) vLLM/OpenAI -->

```
사용자 ──▶ portal(:8080) ──▶ 로컬 LLM (별도 GPU 머신, vLLM)
                        └──▶ OpenAI API
```

**같은 프로세스에 공존하기엔 성격이 너무 달랐다:**

| | 일반 API (로그인·조회) | LLM 추론 |
|---|---|---|
| 응답 시간 | ms 단위 | 최대 120초 |
| 실행 위치 | 앱 서버 | 별도 GPU 머신 / 외부 API |
| 비밀키 | 없음 | `OPENAI_API_KEY`, GPU 서버 주소 |
| 안정성 | 우리가 통제 | 벤더·모델이 죽거나 바뀜 |

     본체 코드에 들어가 있던 것 (현재는 ai-app KwakAiClient.java:21-29)

## T — 무엇을 지켜야 했나

<!-- 성공 기준을 조건으로 명시. R장에서 이 4개를 1:1로 채점한다 -->

분리를 하든 안 하든, 지켜야 할 조건을 먼저 정리했다:

1. **장애 격리** — AI(GPU 서버, LLM)가 죽거나 느려져도 본체 API는 영향 없을 것
2. **응답성** — 사용자는 어떤 경우에도 LLM 추론(최대 120초)을 직접 기다리지 않을 것
3. **변경 격리** — LLM 벤더/모델 교체 시 본체는 수정·재배포 없을 것
4. **시크릿 경계** — AI 관련 비밀키·주소는 본체 환경변수에서 제거할 것

## A — 어떻게 갈랐나

### A-1. 프로세스 분리: ai-app (:8090)

<!-- ai-app의 규칙 3개 강조 -->

LLM 추론"만" 대신해 주는 서버를 하나 뗐다. 규칙은 셋:

- **외부 비노출** — 게이트웨이에 라우트 자체가 없다 (사용자가 닿을 방법이 없음)
- **X-System-Key 인증** — 서버 환경변수에만 있는 키. 브라우저는 못 부른다
- **DB 없음** — 데이터는 모른다. "질문 → 답"만 한다

### A-2. 계약 단일화: AiGatewayClient

  common/src/main/java/kwak/common/ai/AiGatewayClient.java
  "(system, user) 넣으면 content 나온다" — 벤더 상세(한자 버그 프롬프트,
  dummy 헤더, usage 파싱)는 전부 ai-app 안으로 숨었다 -->

### A-3. 시간 분리: Kafka 비동기

<!-- 시퀀스 그림 + "왜 바로 안 주는가" 설명 -->

```
[사용자]                [portal]                      [ai-app]      [GPU/vLLM]
   │ POST /build-context   │                              │              │
   │──────────────────────▶│ Redis 락 + Kafka 발행         │              │
   │◀── 202 "만드는 중" ────│                              │              │
   │                       │ (리스너 스레드가 큐에서 꺼냄)    │              │
   │                       │ 데이터 수집·프롬프트 조립        │              │
   │                       │─────── X-System-Key ────────▶│─── 추론 ────▶│
   │ GET /build-status     │◀──────── 결과 ────────────────│◀─────────────│
   │◀── DONE ──────────────│ 파싱·DB 저장                  │              │
   │ GET /results          │                              │              │
```

- 202를 즉시 주고 폴링시키는 이유: 120초를 사용자에게 기다리게 할 수 없다
- `concurrency = 1`: LLM 추론은 앱 전체에서 한 번에 하나 — GPU 서버 보호(백프레셔)
